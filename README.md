# SPD Beta-Nu — Astro rebuild

The new Sigma Phi Delta Beta-Nu (Cal Poly SLO) site. Astro + Tailwind, designed for Vercel with AI-SEO infrastructure baked in.

## Local dev

```bash
npm install
npm run dev         # http://localhost:4321
npm run build       # static output to ./dist
npm run preview     # serve the built site locally
```

## AI SEO

Audit score progression: original Jekyll site **36/100** → this build (localhost) **88/100**, with the remaining three checks (Agent UA → markdown, Accept header → markdown, Missing page → markdown) all handled at the edge by Vercel rewrites and expected to pass post-deploy.

Implemented:

- `/llms.txt` — chapter overview, links to all `.md` mirrors, facts block (root-relative URLs so it resolves on any host)
- `/robots.txt` — explicit `Allow: /` for GPTBot, ChatGPT-User, OAI-SearchBot, ClaudeBot, Claude-Web, anthropic-ai, CCBot, Google-Extended, PerplexityBot, Perplexity-User, Applebot-Extended, Bytespider, Meta-ExternalAgent, cohere-ai, YouBot
- `/sitemap.xml` — hand-rolled urlset returning the four content pages; the auditor reads this directly to pick sample URLs
- `/sitemap-index.xml` — auto-generated by `@astrojs/sitemap`
- `/sitemap.md` — human / agent-readable index
- `/{page}.md` — markdown mirror for every page (index.md, about.md, recruitment.md, brothers.md, contact.md) with `canonical_url` + `last_updated` frontmatter keys
- `/404.md` — 200-response markdown index served for unknown paths (AI agents discard 404 bodies)
- JSON-LD on every page — `Organization` schema with `parentOrganization`, `memberOf` CollegeOrUniversity, addresses, sameAs, knowsAbout — plus a `WebPage` schema per route
- `<link rel="alternate" type="text/markdown" href="{page}.md">` on every page
- `<meta name="description">` + `<link rel="canonical">` + OpenGraph + Twitter on every page
- `vercel.json` rewrites — when `Accept: text/markdown` is set OR the User-Agent matches a known AI crawler, requests to `/`, `/about`, etc. are transparently rewritten to the matching `.md` mirror; unmatched paths fall through to `/404.md`
- `Vary: Accept, User-Agent` on every response
- Markdown responses return `Content-Type: text/markdown; charset=utf-8`

### Local audit testing

The `@vercel/agent-readability` auditor reads `<loc>` URLs out of `/sitemap.xml` and follows them — so a local audit needs the sitemap to point at localhost. Set `SITE_URL` before building:

```bash
SITE_URL=http://localhost:4321 npm run build
SITE_URL=http://localhost:4321 npm run preview &
npx @vercel/agent-readability audit http://localhost:4321
```

For the canonical production build, leave `SITE_URL` unset — `site.url` defaults to `https://spdcalpoly.com`.

## Deploy to Vercel

```bash
vercel        # first time, link the project (repository root = Astro root)
vercel --prod # ship
```

In the Vercel project settings, set **Root Directory** to **`.`** (repository root), not `web`. Point **spdcalpoly.com** DNS at Vercel when you cut over.

The previous Jekyll site is archived under **`.legacy/`** (optional local builds only).

## Maintenance

Editing chapter content:
- **Officers / exec board** → `src/data/officers.ts`
- **Pledge classes** → `src/data/pledge-classes.ts`
- **Rush schedule + eligible majors** → `src/data/rush.ts`
- **Brother roster** → `src/data/brothers.ts`
- **Global site info** → `src/data/site.ts`

Editing copy:
- Each page body is in `src/pages/*.astro`
- Markdown mirrors are generated from the same data + bodies in `src/lib/page-markdown.ts` — update both when you change the page content (or refactor to share a single source of truth later).

Photos: drop them under `assets/images/img_YEAR/`, then reference `/img_YEAR/file.jpg` in the Astro pages. Symlinks at `public/img_2022` and `public/img_2023` expose them at build time.
